## 2. 과제 개요

본 과제는 수업에서 학습한 SBB 게시판 구조를 바탕으로, **본인 영어 이름을 반영한 새로운 Spring Boot 프로젝트**를 직접 생성하고 게시판 기능을 처음부터 구현한 것입니다.

기존 `sbb` 프로젝트를 그대로 복사하지 않고, 프로젝트명·패키지명·화면 제목을 모두 본인 이름에 맞게 변경하였습니다. 게시판 주제는 **반도체 장비 Q&A**로 설정하였고, 카페·블로그처럼 **회원가입·로그인·등급(일반회원/관리자)** 을 적용하여 실제 커뮤니티와 유사하게 동작하도록 개선하였습니다.

## 3. 사용한 기술

| 구분 | 기술 |
|------|------|
| 언어 | Java 17 |

## 5. 구현한 기능 목록

### 5.1 필수 기능 (지시서 기준)

| 번호 | 기능 | URL | 설명 |
|------|------|-----|------|
| 1 | 게시글 목록 조회 | `GET /question/list` | 번호, 제목, 작성자, 작성일, 답변 수, 상태 표시 |
| 2 | 루트 접속 | `GET /` | 게시글 목록으로 자동 이동 |
| 3 | 게시글 상세 조회 | `GET /question/detail/{id}` | 제목, 내용, 작성일, 답변 목록 표시 |
| 4 | 게시글 등록 | `GET/POST /question/create` | 로그인 후 작성, DB 저장, 목록 이동 |
| 5 | 게시글 수정 | `GET/POST /question/modify/{id}` | 작성자 또는 관리자만 수정 가능 |
| 6 | 게시글 삭제 | `GET /question/delete/{id}` | 확인창 후 삭제, cascade로 답변 함께 삭제 |
| 7 | 답변 등록 | `POST /answer/create/{id}` | 로그인 후 답변 저장 및 화면 출력 |

### 5.2 MVC 구조

```
브라우저 → Controller → Service → Repository → H2 DB → Thymeleaf 화면
```

### 5.3 데이터베이스 관계

| 테이블 | 관계 | 설명 |
|--------|------|------|
| MEMBER | 회원 | 아이디, 비밀번호(BCrypt), 닉네임, 등급(USER/ADMIN) |
| QUESTION | 질문 | Member(작성자)와 N:1 관계 |
| ANSWER | 답변 | Question과 N:1, Member(작성자)와 N:1, hidden(숨김) |
| Question → Answer | 1:N | `@OneToMany(cascade = REMOVE)` — 질문 삭제 시 답변 함께 삭제 |

### 5.4 회원·등급 기능 (추가 구현)

| 기능 | URL | 설명 |
|------|-----|------|
| 회원가입 | `GET/POST /member/signup` | 일반회원(USER) 가입 |
| 로그인 | `GET/POST /member/login` | Spring Security 폼 로그인 |
| 로그아웃 | `POST /member/logout` | 세션 종료, 목록으로 이동 |
| 등급 구분 | USER / ADMIN | 일반회원 / 관리자 |

### 5.5 등급별 권한

| 동작 | 비회원 | 일반회원(USER) | 관리자(ADMIN) |
|------|--------|----------------|---------------|
| 목록·상세 보기 | ✅ | ✅ | ✅ |
| 질문·답변 작성 | ❌ | ✅ | ✅ |
| 본인 글/답변 수정·삭제 | ❌ | ✅ | ✅ |
| 타인 글/답변 수정·삭제 | ❌ | ❌ | ✅ |
| 답변 숨김/숨김해제 | ❌ | ❌ | ✅ |

### 5.6 추가 커뮤니티 기능

| 기능 | 설명 |
|------|------|
| 답변 수정·삭제 | 작성자 또는 관리자 |
| 답변 숨김(블라인드) | 관리자 전용, 공개 답변 수에 반영 |
| 상태 뱃지 | 답변대기 / 답변완료 |
| 통계 | 전체 글, 답변 대기, 전체 공개 답변 수 |
| 에러 페이지 | 403(권한 없음), 404(데이터 없음) 안내 화면 |
| 샘플 데이터 | 서버 최초 실행 시 반도체 Q&A 테스트 글 자동 생성 |

## 9. 구현하면서 어려웠던 점

1. **MVC 계층 연결** — Controller, Service, Repository 역할 분리를 이해하며 구현하였습니다.

2. **Thymeleaf 템플릿 상속** — `layout.html`과 `th:replace` 문법 적용 시 화면 구조를 맞추는 데 시간이 걸렸습니다.

3. **질문 삭제 시 답변 연동** — `@OneToMany(cascade = CascadeType.REMOVE)`로 질문 삭제 시 답변도 함께 삭제되도록 설정하였습니다.

4. **Spring Security 적용** — 로그인, 회원가입, 등급별 권한, CSRF 토큰, 로그아웃 POST 처리를 학습하며 구현하였습니다.

5. **H2 DB 잠금 오류** — 서버 중복 실행 시 DB 파일 잠금이 발생하여 `AUTO_SERVER=TRUE` 옵션을 추가하고, 스키마 변경 시 DB를 재생성하여 해결하였습니다.

6. **8080 포트 충돌** — 터미널과 Eclipse에서 서버를 동시에 실행하면 포트 충돌이 발생하여, 한 곳에서만 서버를 실행하도록 관리하였습니다.
